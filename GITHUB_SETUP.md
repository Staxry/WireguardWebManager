# Инструкция по публикации проекта на GitHub

## Шаг 1: Проверка файлов перед публикацией

Перед публикацией убедитесь, что следующие чувствительные файлы **НЕ** будут загружены:

- ✅ `data/api_keys.json` - исключен через .gitignore
- ✅ `data/wireguard_web.db` - исключен через .gitignore
- ✅ `data/params` - исключен через .gitignore
- ✅ `.admin_config.json` - исключен через .gitignore
- ✅ `logs/audit.log` - исключен через .gitignore
- ✅ Все файлы `*.conf` из директории `wireguard/` - исключены через .gitignore

## Шаг 2: Инициализация Git репозитория

Если Git еще не инициализирован, выполните:

```bash
cd /Users/stax/PycharmProjects/VPNServerControl
git init
```

## Шаг 3: Проверка статуса Git

Проверьте, какие файлы будут добавлены в репозиторий:

```bash
git status
```

Убедитесь, что чувствительные файлы не отображаются в списке файлов для коммита.

## Шаг 4: Добавление файлов

Добавьте все файлы проекта:

```bash
git add .
```

Проверьте еще раз статус:

```bash
git status
```

## Шаг 5: Первый коммит

Создайте первый коммит:

```bash
git commit -m "Initial commit: WireGuard Web Management Interface"
```

## Шаг 6: Создание репозитория на GitHub

1. Перейдите на https://github.com/new
2. Укажите имя репозитория (например, `VPNServerControl`)
3. Выберите публичный или приватный репозиторий
4. **НЕ** добавляйте README, .gitignore или LICENSE (они уже есть в проекте)
5. Нажмите "Create repository"

## Шаг 7: Подключение к удаленному репозиторию

После создания репозитория на GitHub, выполните:

```bash
git branch -M main
git remote add origin https://github.com/<ваш-username>/VPNServerControl.git
```

Замените `<ваш-username>` на ваш GitHub username.

## Шаг 8: Отправка кода на GitHub

Отправьте код в репозиторий:

```bash
git push -u origin main
```

Если потребуется аутентификация, используйте Personal Access Token вместо пароля.

## Шаг 9: Проверка на GitHub

Откройте ваш репозиторий на GitHub и убедитесь, что:
- ✅ Все файлы загружены
- ✅ Чувствительные файлы отсутствуют
- ✅ README.md отображается корректно
- ✅ LICENSE файл присутствует

## Дополнительные настройки (опционально)

### Настройка описания репозитория

На странице репозитория нажмите "Settings" → "General" и добавьте описание:
```
Современный веб-интерфейс на Flask для управления WireGuard VPN-сервером с акцентом на безопасность, аудит и удобство администрирования.
```

### Добавление тегов (Topics)

На странице репозитория добавьте теги:
- `wireguard`
- `vpn`
- `flask`
- `python`
- `web-management`
- `security`

### Настройка защиты ветки main

Рекомендуется настроить защиту ветки main:
1. Перейдите в Settings → Branches
2. Добавьте правило для ветки `main`
3. Включите "Require pull request reviews before merging"

## Важные замечания

⚠️ **ВНИМАНИЕ**: После публикации проверьте, что:
- Нет реальных API ключей в репозитории
- Нет паролей или секретных ключей
- Нет баз данных с реальными данными
- Нет логов с чувствительной информацией

Если вы случайно загрузили чувствительные данные:
1. Немедленно удалите их из истории Git (используйте `git filter-branch` или BFG Repo-Cleaner)
2. Смените все скомпрометированные ключи и пароли
3. Рассмотрите возможность удаления и пересоздания репозитория

## Полезные команды Git

```bash
# Просмотр изменений
git status
git diff

# Отмена изменений в рабочей директории
git checkout -- <файл>

# Просмотр истории коммитов
git log

# Создание новой ветки
git checkout -b feature/new-feature

# Переключение между ветками
git checkout main
```
